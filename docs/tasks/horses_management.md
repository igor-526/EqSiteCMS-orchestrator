# Контекст
У нас на бэкэнде есть следующие эндпоинты:
- **GET** `/api/horses`
- **POST** `/api/horses`
- **GET** `/api/horses/{slug_or_id}`
- **PATCH** `/api/horses/{horse_id}`
- **DELETE** `/api/horses/{horse_id}`
# Backend
Проверь стандартную (когда в sort ничего не передано) сортировку списка лошадей
По умолчанию она должна быть следующей (приоритет):
1. updated_at DESC
2. created_at DESC
Передача любого аргумента в sort должна переопределять это поведение
# Frontend CMS
## Описание
Нам нужно сделать UI для управления лошадьми
UI должен быть выполнен в табличном представлении
Таблица должна быть со следующими колонками
В идеале зафиксировать при горизонтальной прокрутке столбцы "База" и "Кличка"
## Колонки

| Столбец  | Поле                  | Дополнительно                              |
| -------- | --------------------- | ------------------------------------------ |
| База     | this_stable           |                                            |
| Кличка   | name                  | выводим максимум 32 символа                |
| Описание | description           | выводим максимум 32 символа                |
| Порода   | breed.short_name      | fallback name, выводим максимум 32 символа |
| Масть    | coat_color.short_name | fallback name, выводим максимум 32 символа |
| Тип      | kind                  | Русифицировать                             |
| Пол      | sex                   | Русифицировать                             |
| Владелец | horse_owner.name      | выводим максимум 32 символа                |
| Возраст  | age                   |                                            |
| Путь URL | slug                  |                                            |
| Действия |                       | Логика описана ниже                        |
## Фильтрация
Должна быть поддержана фильтрация по всем доступным в API фильтрам
Если для фильтра есть соответствующая колонка в таблице, необходимо реализовать фильтр прямо в таблице
Если такой колонки нет, то фильтры выносятся селектами или input слева от панели пагинации
По умолчанию должен быть применён фильтр `this_stable = true`
Некоторые фильтры (например, масти, породы или владельцы) требуют предварительной загрузки значений по API
Референс загрузки фильтров по API: раздел "Галерея", фильтр по услугам
Референс таблицы: раздел "Услуги и цены", вкладка "Цены и услуги"
## Сортировка
Необходимо поддержать доступную на backend сортировку только с пересечением столбцов. То есть вся сортировка должна быть реализована только в столбцах. Ничего никуда не выносим, ничего дополнительного не поддерживаем. Строго пересечение столбцов и поддерживаемой на backend сортировки
## Действия
В колонке действия должны располгаться кнопки
Референс: раздел "Лошади", вкладка "Породы"
### Фотографии
Используй иконку, которая уже использовалась для фотографий
Логику при нажатии скопируй, например, из раздела "Услуги и цены", вкладка "Цены и услуги"
Рядом с иконкой расположи маленькой цифрой количество привязанных фотографий. Можно взять из DTO
### Родословная
Кнопка должна открывать модальное окно работы с родословной. Пока что в модальном окне выведи просто, что функционал ещё в разработке
Подбери для кнопки подходящую иконку
Рядом с иконкой мне нужно три точки
1. Серая или синяя (синяя при наличии `pedigree.sire`)
2. Серая или розовая (розовая при наличии `pedigree.dam`)
3. Серая или зелёная (синяя при наличии `pedigree.foals`)
При наведении на иконку должна быть аннотация в следующем виде
```plaintext
Отец: {pedigree.sire.name}
Мать: {pedigree.dam.name}
Потомство: {pedigree.foals.name перечисление if <= 2 else len(pedigree.foals)}
```
## Добавление и обновление лошади
Посмотри логику добавления и обновления записи на соседних вкладках
Так как форма добавления большая, там будут нюансы
### Категоризация
Нам нужно разделить форму на категории. Каждую категорию отметь отдельным заголовком (не броским, не большим)
**Основные данные**
- name
- description
- kind (селектор)
- this_stable (селектор)
- sex (селектор)
**Дополнительно**
- breed_id
- coat_color_id
- height
**Даты**
- bdate
- bdate_mode
- ddate
- ddate_mode
**Владелец**
- horse_owner_id

В разделе основные данные расположи в одну линию kind, this_stable и sex
В разделе дат расположи селектор даты и режим отображения попарно у каждой из дат в одну линию
## Расположение
Должно располагаться в разделе "Лошади", вкладка "Лошади"
Вкладка должна быть первой в списке и открываться по умолчанию
## Инструкция и документация
Посмотри как в разделе "Услуги и цены" или "Новости" реализованы вкладки инструкций
Суть в том, что одна из них для разработчиков, вторая для администраторов для использования
Там есть пример организации видимости этих вкладок в зависимости от роли пользователя
Нам нужно составить подробную инструкцию для всех вкладок этой страницы
- Лошади
- Породы
- Масти
- Владельцы
- Услуги
