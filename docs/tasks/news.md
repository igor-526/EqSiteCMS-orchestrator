# Контекст
Нам необходимо реализовать раздел новостей для CMS
# Путь пользователя
Администратор в CMS может добавлять новости, которые потом высвечиваются на сайте
# Требования
#### Дата новости
Дата новости не совпадает с created_at или updated_at. Нужно отдельное поле published_at
#### Soft delete
Давай реализуем удаление новостей
#### Атрибуты
- Наименование (63)
- Сниппет (255 nullable). Автогенерируемый в случае с null
- Содержимое (HTML)
- Фотографии
#### Фотографии
У новости могут быть фотографии из галереи. Нужно построить связь аналогичную с prices. С выбором нескольких фотографий и выбором главной
#### Сниппет
Сниппет может быть автоматически сгенерирован, если не задан явно. Сниппет хранится как только текст. В CMS UI в ячейке сниппет обрезается до 32-х символов.
# Доступы
- superuser
- admin
- dev
# Backend
#### Soft Delete
Давай реализуем отдельным миксином по аналогии с timestamp следующие колонки:
- is_deleted: bool
- deleted_at: datetime
#### GET Endpoints
Нам нужны 2 разных эндпоинта. Первый для CMS, второй для consumers. CMS эндпоинт видит новости из будущего, удалённые и тд. Consumer эндпоинт автоматически фильтрует по published_at только выпущенные новости и которые не удалены
# Frontend CMS
В главном меню нужно добавить раздел новостей, которые будут оформлены по аналогии с другими разделами. Представляем всё в табличном виде
Для редактирования новости используй текстовый редактор, который был реализован в docs/reports/FEATURE-page-data-editor-review.md
