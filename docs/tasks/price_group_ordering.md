# Проблема
У нас есть модели услуг и групп услуг
services/backend/src/models/prices.py
У каждой группы может быть много услуг, как и у каждой услуги может быть много групп
Проблема в том, что нам нужно давать возможность задавать конкретный порядок этих услуг в рамках одной группы. Это должно быть сортировкой по умолчанию, если никакая другая не задана

# Agents
Давай в этой итерации ещё доработаем агента backend
Я хочу явно там прописать следующие правила для реализации всех display_order
1. Все reorder происходят отдельными POST эндпоинтами, не отдельными PATCH элемента в списке
2. display_order элементов может находиться исключительно с диапазоне 1 <= do <= len(elements)
3. display_order не передаётся явно в OutDto, frontend всегда высчитывает его сам
4. в эндпоинтах на reorder frontend шлёт не весь список, а только список изменений. Например,
```json
[
{
	"id": "UUID элемента 1",
	"order": 3
},
{
	"id": "UUID элемента 2",
	"order": 4
}
]
```

	1. Сначала элемент 1 встаёт на место 3. Если это место занято (а оно будет занято), то все остальные элементы смещаются на 1 вперед. Нужно учитывать порядок всех элементов, чтобы они не выходили за допустимые пределы
	2. Аналогичное происходит с элементом 2
5. При работе с reorder каждый новый элемент при воздействии других эндпоинтов всегда встаёт на последнее место
6. При работе с reaorder нужно уделить особое внимание другим эндпоинтам, которые взаимодействуют с элементами. Особое внимание unit и smoke тестам
7. Очень частая ошибка, которую нужно избежать заранее - ограничения уникальности. Следует делать display_order колонку nullable и обновлять display_order всегда в 2 итерации. Сначала сброс на null, потом присвоение новых значений
