# Проблема
Сейчас проект заточен под то, что на каждую конюшню мы размножаем копии CMS. Это неудобно для масштабирования. Я предлагаю использовать один экземпляр backend и frontend для CMS для разных конюшен
Для этого я предлагаю добавить корневую сущность Equestrian
А с этой сущностью уже будут связываться другие элементы

Следующие модели:
- services/backend/src/models/breeds.py
- services/backend/src/models/coat_color.py
- services/backend/src/models/horse_owner.py
- services/backend/src/models/horse_service.py
- services/backend/src/models/horse.py
- services/backend/src/models/photos.py
- services/backend/src/models/prices.py
- services/backend/src/models/site_settings.py
Не имеют никакой связи с пользователем, так как задумка была в том, что в рамках одной конюшни все сущности общие и могут быть созданы/отредактированы/добавлены/удалены любым пользователем на основании доступов
Давай свяжем их с конюшней

services/backend/src/models/users.py
Пользователь нужен только для авторизации. У него нет никаких связей с другими сущностями. Давай тоже свяжем с конюшней
