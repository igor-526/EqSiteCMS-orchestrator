# Проблема
В системе авторизации CMS фронтенда что-то сломано. Все запросы с access токеном проходят без проблем, авторизация происходит без проблем, но вот когда он протухает, возникает следующая проблема:
Запрос из CMS (любой) идёт без access токен, только refresh (в куках)
Backend возвращает не 401-ю ошибку, а 400 с содержимым:
```json
{"detail":"Отсутствует X-Equestrian-Service-Key"}
```
Правильный порядок действий должен быть следующий:
1. access токен протух / запрос идёт только с refresh токеном
2. backend возвращает 401
3. frontend делает refresh запрос
	1. Если refresh не протух:
		1. Backend выставляет новый access токен в cookies
		2. Fronted переделывает запрос
	2. Если refresh протух
		1. Backend возвращает 401
		2. Frontend делает редирект на страницу логина
# Задача
Планируй изменения только в backend и в CMS frontend зонах (в зависимости от источника проблемы)
Напиши в docs/plans/bugfix план по исправлению данного бага
